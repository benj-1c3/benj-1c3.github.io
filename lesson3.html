Java Basics

1. Java Syntax

Java syntax defines the structure and rules for writing programs. Java is strictly typed, meaning every variable and method must be declared with a specific type.

Main Features of Java Syntax:

  • Classes and Objects: Every Java program is written inside a class In Java, a class is a blueprint for creating objects. It encapsulates data for the object (attributes or fields) and methods to manipulate that data (functions or behaviors). Here are a few key points:

    • Blueprint: Think of a class as a recipe or template that defines what an object will be. It outlines the properties (data) and actions (methods) that the object can have.


    • Encapsulation: A class bundles data and behavior together, promoting better organization and modularity in code.


    • Instantiation: You create objects (instances) from a class. Each object can have its own values for the attributes defined in the class.


    • Inheritance: Classes can inherit properties and methods from other classes, allowing for hierarchical classification and code reuse.

      • for example:


      public class Car { // Fields (attributes) String color; String model; // Method (behavior) void startEngine() { System.out.println("Engine started!"); } } // Creating an instance of Car public class Main { public static void main(String[] args) { Car myCar = new Car(); myCar.color = "Red"; myCar.model = "Sedan"; myCar.startEngine(); // Output: Engine started! } }

      In this example, the Car class defines the structure of a car object, while the Main class demonstrates how to create and use an instance of Car.


  • Curly Braces `{}`: Define code blocks.
    • In Java, a block is a group of statements enclosed within curly braces {}. Blocks are used to define the scope and grouping of code. Here are some key points:

    • Scope Definition: Variables declared inside a block are only accessible within that block.

    • Control Structures: Blocks are commonly used with control structures like if, for, while, and try-catch to group multiple statements.

    • Method Bodies: The code inside a method is also a block, defining what the method does.

      • public class BlockExample { public static void main(String[] args) { // This is a block within the main method { int x = 10; System.out.println("Inside the block, x = " + x); } // x is not accessible here as it was declared inside the block above } }

      In this example, the curly braces create a block that limits the scope of the variable x to that block only.

  • Semicolons `;`: End statements. In Java, the semicolon (;) is used to signal the end of a statement. Think of it as a period at the end of a sentence in written language—it tells the compiler where one instruction ends and another begins. Here are some important points:

    • Statement Termination: Every executable statement, such as variable declarations, assignments, or method calls, must end with a semicolon.

    • Separation of Statements: Semicolons clearly separate one instruction from another, ensuring the code is parsed correctly.

    • Syntax Errors: Omitting a semicolon where required will result in a syntax error, preventing the code from compiling.

      • for example:

      int a = 5; // Variable declaration and initialization a = a + 10; // Assignment operation System.out.println(a); // Method call to print the value of 'a'

      Without the semicolons, the Java compiler wouldn’t be able to determine where one statement ends, leading to errors.

  • Case-Sensitive: Java is case-sensitive, meaning variableName and VariableName are treated as different variables.

      • for example

      int myNumber = 10; int MyNumber = 20; System.out.println(myNumber); // Prints 10 System.out.println(MyNumber); // Prints 20

  • Main Method: Entry point of every Java application.
    • The entry point of every Java application.
    • It must always have the following signature:
      • public static void main(String[] args)

    • This method is executed when the program starts.

      • example:

      public class MainMethodExample { public static void main(String[] args) { System.out.println("Java programs start execution from the main method."); } }

    • Without this method, the program won’t run.

2. Variables & Data Types

A variable is a container for storing values, and a data type defines the kind of data it can hold.

Primitive Data Types:

  • int: Stores whole numbers (e.g., 10, -5).
  • double: Stores floating-point numbers (e.g., 3.14, 99.99).
  • char: Stores a single character (e.g., 'A', 'Z').
  • boolean: Stores true or false values.
int number = 10; double price = 99.99; char letter = 'A'; boolean isJavaFun = true;

3. Operators

Operators perform operations on variables and values:

  • Arithmetic: +, -, *, /, % (modulus)
  • Comparison: ==, !=, >, <, >=, <=
  • Logical: && (AND), || (OR), ! (NOT)

4. Control Flow (if-else, loops)

Control flow structures allow decision-making and repetition.

if (number > 0) { System.out.println("Positive number"); } else { System.out.println("Negative number"); }
for (int i = 0; i < 5; i++) { System.out.println("Iteration: " + i); }

5. Object-Oriented Programming (OOP)

Java follows OOP principles like:

  • Encapsulation: Restricting access to data within a class.
  • Inheritance: One class inherits another’s properties.
  • Polymorphism: Methods behave differently across classes.

6. Exception Handling

Exceptions handle errors using try-catch blocks.

try { int result = 10 / 0; } catch (ArithmeticException e) { System.out.println("Cannot divide by zero!"); }

7. Input/Output Handling

Java allows user input using the Scanner class.

import java.util.Scanner; Scanner sc = new Scanner(System.in); System.out.println("Enter your name:"); String name = sc.nextLine(); System.out.println("Hello, " + name);
